======================================================================
1. PROJECT IDENTITY
======================================================================

Repository: cag-llm-kvcache
Full name: CAGE (Cache-Augmented Generation Evaluation)
Author: Lucas Mariano do Carmo (PUC Minas)
Contact: lucas.mariano.carmo@gmail.com

CAGE is a benchmarking/evaluation framework for comparing cache-based LLM serving
strategies against retrieval-based (RAG) approaches. It is NOT a serving engine.
vLLM is the serving engine. CAGE orchestrates experiments, measures metrics, and
evaluates quality via HTTP communication with vLLM.

4. CRITICAL FACTS
======================================================================

- CAGE communicates with vLLM only via HTTP API. It does NOT import vLLM Python code.
- The vLLM/ directory is vendored for reference only. The experiment runner ignores it.
- All experiment data is self-contained in analysis/phase1/results/.
- The paper uses SBC template format (sbc-template.sty). Do not override template settings.
- Prefix Cache and No Cache share identical quality metrics (both use gold passages).
- BERTScore is non-discriminative (0.324-0.328). Faithfulness is the primary quality metric.
- The Distributed baseline had 0 actual KV transfers — it only tests prefix-hash routing.

======================================================================
5. KEY VALIDATED NUMBERS (Phase 1)
======================================================================

Performance vs No Cache baseline:
- Prefix Cache: -37.4% latency, -65.7% TTFT, +58.3% QPS
- RAG: +70.4% latency, +171.4% TTFT, -45.1% QPS
- Hybrid Warm: -17.1% latency, -59.7% TTFT
- Distributed: -23.7% TTFT, +15.5% latency overhead, 7.6x p95/p50 TTFT spread

Quality:
- No Cache / Prefix Cache faithfulness: 0.570 (identical)
- RAG faithfulness: 0.504 (-11.6%)
- Distributed faithfulness: 0.636 (highest but high variance)
- Prompt-cached ratio: Prefix Cache 68.4%, Hybrid Warm 89.2%
